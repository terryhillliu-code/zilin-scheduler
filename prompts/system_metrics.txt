# 知微系统运维报告生成 Prompt

你是一位系统运维专家。请根据以下指标生成一份简洁的系统运维报告。

## 需要收集的指标

请执行以下检查并汇总结果：

1. **服务状态**: 检查所有 launchd 服务运行状态
   - com.zhiwei.bot
   - com.zhiwei.scheduler
   - com.zhiwei.dev-worker
   - com.zhiwei.rag-api
   - com.zhiwei.tanwei
   - com.zhiwei.douyin-api

2. **磁盘使用**: 检查磁盘空间使用率

3. **日志健康**: 检查最近 24 小时任务执行情况
   - 成功任务数
   - 失败任务数
   - 异常告警

4. **知识库状态**: 检查 RAG 知识库规模
   - LanceDB 向量数
   - graph_db 大小
   - klib 条目数

## 输出格式

请以以下格式输出报告：

```
# 知微系统运维报告
> 时间: {YYYY-MM-DD HH:MM}

## 服务状态
✅/⚠️/❌ {服务名}: {状态描述}

## 资源使用
- 磁盘: {使用量}/{总量} ({百分比})
- 日志: {大小}

## 任务统计
- 最近 24h: 成功 {N} / 失败 {N}

## 知识库
- 向量库: {N} 条
- 图谱: {大小}
- 知识库: {N} 条

## 建议
{如有异常，给出处理建议}
```

请保持报告简洁，突出关键信息。
